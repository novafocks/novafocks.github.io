Introducing C and Pointers

C, despite how old it is, seems to have aged decently enough. After all, it powers a massive portion of the kernel of the OS which I'm using to type this blog. That, of course, might change given Rust's introduction into the Linux kernel as an easier solution to implement drivers and stuff, but that's a topic for another day. C works incredibly close to the hardware and is easy enough for someone to learn by itself, granted you don't make any stupid mistakes in memory allocation. When working with audio, this means that it is quite efficient but also practical to code, which are two important facets of working with audio.

However, one of the technologies most used in C and C++ is the pointer system, which I struggled to wrap my mind around before looking into it some. So before getting into how C works with audio directly, I just wanted to demonstrate how pointers work on a fundamental level, and what better way than with a simple example? Fair warning that you should be familiar with the basics of C syntax.

#include <stdio.h>

int main() {
 int x = 15;
 printf("The value of x is %d\n", x);

 int *ptr = &x;
 printf("The pointer of x is %d\n", ptr);

 int y = *ptr;
 printf("The value of y, dereferenced from the pointer, is %d\n", y);
}

So let's break this down line by line, leaving behind importing the standard library and creating the main function. First we created an integer variable x, assigned the value of 15. We then printed it out using the printf() function from the C standard library. That's self-explanatory. Where it becomes particularly tricky is when we start playing with the pointers.

We declared a variable ptr, but there was an asterisk or star (*) next to it. This designates that the variable is a pointer, and the type given that would otherwise declare the variable, is the type of variable that the pointer points to.

So what exactly is a pointer? It's essentially the address in memory for a variable, which every single variable has. Think of the variable x as a container in a warehouse. The contents of the container are the value of the variable, in this case the integer 15. But in a warehouse, you have a lot of containers, so you need an easy way to label them. So let's say they create spaces designated for containers, labeled 0 to 255. The containers placed in these spaces may change, but the spaces themselves do not. Let's say that the variable x is stored in space 129. If the value stored inside the container is 15, then the pointer for the container is 129. I'm not sure if this is 100% accurate to how computers work fundamentally, there might be a few issues with that analogy, but it's a decent metaphor you could visualize.

Then we have an ampersand (&) next to the previously declared x? This designates that the ptr variable isn't going to be assigned the value of the variable x, but the pointer value of the variable x. We'll see how this works when we receive the output. Speaking of, we pass the ptr variable's value into the printf() function as an integer, just as before, in the next line.

Now here's the coolest part. We declare an entirely new and unrelated integer, y. But we're assigning it the ptr variable with another asterisk or star (*). Given that the ptr variable stores the pointer for x, we can reference the variable value using that knowledge. In this case, that asterisk or star designates the value rather than the pointer itself. And from this, we can finally review our output.

The value of x is 15
The pointer of x is -1749144944
The value of y, dereferenced from the pointer, is 15

Everything seems to be in order! The value of the variable x is 15 as it should be, the pointer value references a location in memory, and the value of y can be extracted directly from the information provided by that pointer.
