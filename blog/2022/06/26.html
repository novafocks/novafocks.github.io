So how did setting up my first JUCE plugin go?

First JUCE Plugin

So I installed it and created a new JUCE project (side note, JUCE thought that it should be located in the root folder of the drive by default, which... why?), and it just didn't work. It gave me an error message about how it was incompatible and how some application couldn't be found. As a result, I took the logical course of action and completely reinstalled the program. And that fixes it for some reason, so crisis averted.

So I just followed the tutorial on JUCE for coding your first plugin and made a MIDI volume slider. First I recreated the GUI elements to a T from the tutorial, because what else are you going to do when you first start? And even building it before putting in the actual functionality was a bit confusing at first. It took a simple Google search. But once it was up and running, it just worked.

Well, that's it! My first plugin. Does it do anything? Barely. Is it pulled directly from a tutorial? Of course. But it's a start for certain.

So beyond the "creating a basic audio plugin series", what else resources are there?

For other tutorials on the website there's quite a few categories, including that for creating plugins, synths, DSP (ooh, look, an easier resource for DSP besides an undergraduate textbook!), interface design, and working with MIDI and audio. I could certainly look through these if I need to know any specifics. JUCE also provides a few third-party resources in various locations.

Not to mention, there are plenty of example projects I could use as an introduction. I think the next plugin I'd want to make would be a gain control, just so I could get familiar with procesing real-time audio. Indeed, there is a gain example plugin, and I think analyzing it in a blog post and recreating it from scratch would be a good way to begin. That, or using a tutorial about how to process audio and figuring it out for myself. Which ever way I decide I'd like to.

As for documentation, there's an overview of every class built into the framework alongside their respective methods right here. This will be useful for later, after I'm confident enough to build the foundations of a project myself and need to browse additional features, able to understand what each of them could contribute.
