Linear Systems and the Fundamental Concept of DSP

So what exactly is a linear system in DSP? Remembering that a system deals with signals by taking an input and creating an output as a result, then linear is a property that can be assigned to a system if its results fall into certain parameters. Otherwise, it would be non-linear.

In order for a system to be considered linear, it must have two mathematical properties: homogeneity and additivity. So what do those mean? In order for a system to have homogeneity, then a change in the amplitude of the input signal must correspond to an equal change in amplitude for the output signal. Let's say we have an input of a sine wave into a system, which outputs a cosine wave. If we amplify the input wave by a factor of two, the output wave must also be amplified by a factor of two. This system has homogeneity. If the output wave amplified by a factor of four as a response to an input amplification by a factor of two, then the system would not have homogeneity.

Additivity is a little bit more self-explanatory. Let's say we have two input signals, which we'll call x1[n] and x2[n]. We input these signals into the same exact system, which gives output signals of y1[n] and y2[n] respectively. This system would have additivity if when you input the signal of x1[n] + x2[n], you receive an output which is equal to y1[n] + y2[n].

When a system has these two properties, it is considered a linear system. Examples of this include amplification and attenuation of a signal, convolutions, echoes, and many others. Systems which don't have both of these properties, non-linear systems, include distortion and saturation (including overdriving of analog circuitry), multiplication of a signal by another signal, or systems that include thresholds, such as gates which do not allow output unless the amplitude, for example, has reached a certain level.

Sprinkled into all that jargon was, fun fact, the fundamental concept of DSP. It's called superposition, and it was the foundation of the additivity property I just described; signals can be defined as sums of simpler waveforms, and as such we can use simple signals to understand how signals are changed by systems. This is the foundation of basically every signal processing technique.

Adversely, the concept of decomposition is understanding that complex signals can be created through sums of simpler signals, and using that knowledge to break down said signals. Quite fascinating!

We can use this knowledge to introduce a new concept which is equally fundamental for the field, but I want to introduce it with an example you're likely familiar with.

But convolution is essential to the world of DSP because it builds directly upon superposition and directly applies it: it is the mathematical way to combine two signals to form a third signal.
